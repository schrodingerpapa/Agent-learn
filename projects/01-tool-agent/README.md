# 01-tool-agent

目标：实现一个最小工具调用 Agent。它可以接收用户任务，判断是否需要工具，调用本地工具，并输出带工具调用日志的最终回答。

## 功能范围

第一版只做 CLI，不做 Web 页面。

必须支持：

- 计算器工具：处理简单四则运算。
- 文本搜索工具：从给定文本中查找关键词。
- 时间或文件工具：获取当前时间，或读取并摘要本地文件。
- 工具调用日志：展示 Agent 调用了哪个工具、输入是什么、输出是什么。
- 错误处理：工具参数错误、工具执行失败时，给出可理解的回答。

暂时不做：

- 多 Agent 协作。
- 长期记忆。
- 复杂前端。
- 自动联网搜索。

## 推荐技术栈

- Python 3.11+
- Pydantic：定义工具输入输出。
- pytest：测试工具函数。
- OpenAI API 或 OpenAI Agents SDK：实现工具调用。

## 最小运行目标

用户输入：

```text
帮我计算 23 * 17，并解释一下结果。
```

期望行为：

```text
工具调用：
- calculator(expression="23 * 17")

最终回答：
23 * 17 = 391。
```

## 评估指标

- 工具选择准确率：应该调用工具时是否调用了正确工具。
- 无工具回答准确率：不需要工具时是否没有乱调用。
- 错误处理率：工具失败时是否能给出合理解释。
- 端到端成功率：最终回答是否满足用户任务。

## 面试讲法

可以这样开头：

> 我做了一个最小工具调用 Agent，用来理解 Agent loop 的核心机制。项目重点不是包装聊天接口，而是设计工具 schema、处理工具失败、记录调用日志，并用评估样例验证模型是否能正确选择工具。

需要准备的追问：

- 为什么工具描述会影响模型是否调用工具？
- 工具调用失败后怎么恢复？
- 如何判断 Agent 是“乱调用工具”还是“合理探索”？
- 如果工具很多，如何降低选择错误？
- 这个项目如何扩展成 RAG 助手？
