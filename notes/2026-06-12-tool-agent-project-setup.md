# 工具调用 Agent 项目搭建笔记

日期：2026-06-12

## 我今天要搞懂的问题

- Python 项目为什么要区分 `src/`、`tests/` 和配置文件？
- CLI 程序的最小输入输出流程是什么？
- 工具函数应该如何设计输入、输出和错误返回？
- 在不接入 LLM 的情况下，如何先模拟一个最小 Agent loop？

## 最小 CLI Agent loop

```text
用户输入任务
  -> CLI 接收输入
  -> Agent 根据规则判断是否需要工具
  -> 如果需要，选择工具并准备参数
  -> 执行本地工具函数
  -> 记录工具调用日志
  -> 整合工具结果并输出最终回答
```

## 我的理解

问题1：
  - `src/` 用来放正式代码，避免测试时误导入当前目录里的临时文件；
  - `tests/` 用来放测试代码，验证工具函数和 Agent 行为是否符合预期；
  - `pyproject.toml` 用来记录项目元信息、依赖、测试配置和代码规范。

问题2：
  - CLI 的最小流程是：读取用户输入，调用核心逻辑，打印结果；
  - `cli.py` 不应该写太多业务判断，最好只负责命令行交互；
  - 真正的任务判断、工具选择和结果整合应该放到 `agent.py`。

问题3：
  - 工具函数要能独立运行和独立测试，不依赖模型；
  - 输入参数要少而清晰，例如 `calculator(expression: str)`；
  - 输出结构要稳定，建议包含 `success`、`result`、`error`；
  - 错误输入不能让程序直接崩溃，而是返回可理解的错误信息。

问题4：
  - 规则版 Agent 可以先用关键词判断工具选择，例如“计算”调用计算器；
  - 这样做的目的不是让 Agent 变聪明，而是先跑通完整工程闭环；
  - 等工具、日志、测试都稳定后，再把规则判断替换成 LLM tool calling。
