## 最小 Agent loop

```text
用户输入
  -> 模型判断是否需要工具
  -> 如果需要，生成工具名和参数
  -> 程序执行工具
  -> 工具结果返回给模型
  -> 模型整合并给出最终回答
```

## 我的理解

问题1：
  - agent能够自主进行规划，选择工具，多轮闭环完成任务；
  - 聊天机器人单轮问答，对话理解和文本生成，无工具调用
问题2：
  - 输入请求——>模型决策——>调用工具——>返回结果——>整合输出
问题3：
  - tool schema可以理解为模型调用API的说明书。
  - Agent需要调用外部API、执行搜索或操作数据库，必须以结构化的格式，通常为json schema告诉AI工具如何使用，需要哪些参数以及参数是否必须。
  - schema描述模糊不清会导致工具调用错误，或者参数描述确实错误，工具无法使用，或者结构复杂解析出错
问题4：
  - 工具调用层——调用错误，参数缺失，格式异常；
  - 任务规划层——目标拆解能力弱，步骤顺序混乱，边界判断失误陷入无限循环
  - 记忆与上下文——上下文溢出，记忆失效
  - 提示词引导不足
