# 秋招导向项目清单

选择原则：少做玩具，多做能体现工程判断的项目。建议最终主推 1 个主项目，加 2 个辅助项目。

## 项目 1：工具调用 Agent

定位：入门项目，证明你理解 Agent loop 和 tool calling。

功能：

- 用户输入任务。
- Agent 判断是否需要工具。
- 工具支持计算、文件搜索、网页摘要或本地知识查询。
- 返回结构化结果和工具调用日志。

加分项：

- 工具失败时自动降级。
- 工具调用前做参数校验。
- 为工具选择准备小型 eval。

简历表达：

> 实现基于函数调用的任务型 Agent，设计工具 schema、错误处理和结构化输出机制，并构建测试集评估工具选择准确率与端到端成功率。

## 项目 2：个人知识库 RAG 助手

定位：秋招最稳项目，能覆盖 RAG、后端、评估、部署。

功能：

- 导入 Markdown/PDF/网页笔记。
- 生成向量索引并支持语义检索。
- 回答问题时提供引用来源。
- 无依据时拒答。

加分项：

- rerank、query rewrite、multi-query retrieval。
- 检索和生成分开评估。
- 管理后台展示 trace、top-k 文档、成本和延迟。

简历表达：

> 构建面向个人学习资料的 RAG 问答系统，完成文档切分、向量检索、引用生成和离线评估，针对幻觉和漏检问题优化召回与回答可信度。

## 项目 3：MCP 本地工具服务器

定位：体现你跟得上 Agent 工程生态。

功能：

- 提供 `search_notes`、`get_project_summary`、`create_interview_card` 等工具。
- 用清晰 schema 描述输入输出。
- 支持本地客户端或 Agent 框架调用。

加分项：

- 工具权限分级。
- 审计日志。
- 对提示注入和危险输入做防护。

简历表达：

> 基于 MCP 实现本地工具服务器，将学习资料检索、项目摘要和面试卡片生成封装为可复用工具，并接入 Agent 客户端完成标准化工具调用。

## 项目 4：主项目建议：秋招 Agent 助手

定位：主项目，和你的目标直接相关。

功能闭环：

- 输入目标岗位 JD。
- 自动抽取岗位能力要求。
- 匹配你的项目经历和知识缺口。
- 生成学习计划、简历 bullet、面试追问题。
- 保存每次分析结果，形成求职进度库。

推荐技术栈：

- Backend：FastAPI + SQLite/PostgreSQL + SQLAlchemy。
- Agent：OpenAI Agents SDK 或 LangGraph。
- Retrieval：FAISS/Chroma + rerank。
- Eval：pytest + JSONL case set + LLM-as-judge 或规则检查。
- Observability：结构化日志 + trace 导出。

加分项：

- Human-in-the-loop：简历生成前必须确认事实。
- 多轮状态：记录每个公司的准备进度。
- 安全边界：不编造经历，不输出无法证明的项目成果。

简历表达：

> 设计并实现面向秋招准备的 Agent 助手，支持 JD 能力抽取、个人项目匹配、学习计划生成和面试问题模拟；通过 RAG 引用个人资料，结合规则评估和人工确认降低经历幻觉风险。

## 项目选择建议

如果时间紧：

1. 主攻项目 4。
2. 将项目 2 的 RAG 能力合并进项目 4。
3. 将项目 3 的 MCP 做成一个小亮点。

如果时间充足：

1. 项目 1 用 1 周完成。
2. 项目 2 用 3 周完成。
3. 项目 3 用 2 周完成。
4. 项目 4 用 5-6 周打磨。
